# 守卫监管家长端 Stitch UI 生成提示词

## 使用目的

这份文档用于给 Stitch 生成“守卫监管”家长端 UI 设计稿。当前路线是：应用锁端继续做原生 Android App；家长端先把 PWA 做到接近原生 App 的观感，后续再封装成 Android APK。

生成目标不是网页后台，而是一套可以指导 PWA 改版和后续 APK 壳封装的移动端 App 高保真界面。

## 1. 总提示词

```text
请为“守卫监管”设计一套移动端家长监管 App UI，高保真，适合 PWA 使用，未来会封装成 Android APK。它不是网页后台，而是一个真正像原生 App 的手机界面。

产品定位：
这是家长端 / 监管人端，用来审批孩子手机上的应用放行请求、查看当前放行状态、远程控制被监管手机、查看审批记录和管理账号。

用户感受：
不是冷冰冰的控制工具，而是温和、可靠、克制的家庭守护工具。界面要让人感觉安心、干净、有秩序、有高级感。

整体风格：
温馨家庭守护感 + 极简高级 App + 暖白背景 + 墨绿主色 + 柔和橙色点缀 + 精致真实插画。

视觉要求：
- 背景使用暖白色，不要黄腻。
- 卡片接近白色，边框很淡，阴影柔和。
- 主色使用深墨绿、青绿色。
- 辅助色使用温暖橙色。
- 危险操作使用克制红色。
- 标题可以有一点高级衬线/宋体气质，正文使用清晰现代无衬线字体。
- 图标必须精致，不能用粗糙线框图标。
- 关键视觉请使用高质量插画或图片资产，例如家庭守护插画、手机设备图、账号头像、盾牌手机图、应用图标。
- 不要用 CSS 色块感的简单图形冒充插画。
- 页面必须像真实 App，不要像网页管理后台。

设备尺寸：
请按 390x844 手机竖屏设计。不要画浏览器地址栏，不要出现网址，不要出现网页顶部栏。可以有 App 状态栏和原生 App 顶栏。

底部导航：
登录后固定三项：首页、设备、我的。
未登录页面不显示底部导航。

绝对不要出现：
- DeepSeek 字样
- 网页后台感
- 大面积黄色背景
- 蓝紫科技风
- 赛博风
- 粗糙线框图标
- 太幼稚的卡通风
- 假按钮、假箭头、点了没用的装饰控件
```

## 10. 设备控制页提示词

```text
页面名称：设备控制

顶部标题：设备控制
右上角可以是“记录”或“状态”入口，不要放一个点了没反应的铃铛。

设备卡：
只展示当前选中的一个设备。
包含：
- 更真实的手机设备图片
- 设备名称，例如：红米手机
- 设备型号弱化显示，例如：设备：25060RK16C
- 在线状态胶囊
- 最近操控时间
- 后台保护运行中

如果有多个设备，顶部显示横向设备切换胶囊。
如果只有一个设备，不显示复杂切换区。
不要重复出现两个设备控制台。

控制区使用原生开关卡：
1. 后台保护
说明：控制手机端保护服务，命令由手机后台拉取后生效

2. 高风险入口
说明：保护系统设置、卸载、权限关闭等入口

3. 桌面图标
说明：控制是否显示桌面图标

每个开关都要像真实 App 开关，不要做成两个普通按钮。
辅助操作：立刻重新锁紧，放在低优先级位置。

最近远程命令：
只显示当前设备的最近命令。
列表为左图标、中间文字、右侧状态。
文字不能跑出图标，状态标签不能压住标题。
无命令时显示轻量空状态。

不要出现任何 DeepSeek 字样。
底部导航高亮：设备。
```
