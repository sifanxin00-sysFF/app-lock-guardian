# 学生端联调接口说明

本文件记录学生端 11 个页面第一版可联调接口。它是 mock 联调层，不是最终完整业务系统。

## 基本规则

- 接口前缀：`/api/student/*`
- 返回格式：JSON
- 当前联调版默认免登录，后续正式版再接设备身份、家庭、监管人账号。
- Android 系统权限、已安装应用、真正锁应用能力都只能由学生端本地服务完成；后端只保存最近上报状态和规则。
- 当前 mock 数据使用“示例学生”“示例设备”“示例监管人”等占位身份。

## 绑定监管人

- `GET /api/student/binding/code`
- `POST /api/student/binding/refresh`
- `GET /api/student/binding/status`
- `GET /api/student/binding/qr?code=A7K9Q2`
- `POST /api/student/binding/mock-complete`

`mock-complete` 只用于前端联调，把绑定状态从 `pending` 改成 `bound`。

## 权限

- `GET /api/student/permissions/status`
- `POST /api/student/permissions/report`

`GET` 返回最近一次权限快照。`POST` 预留给 Android 本地服务上报真实权限，不代表后端能直接检测手机权限。

## 首页与守门模式

- `GET /api/student/home/overview`
- `GET /api/student/guard/status`
- `POST /api/student/guard/start`
- `POST /api/student/guard/end`
- `GET /api/student/guard/detail`

`POST /api/student/guard/start` 请求示例：

```json
{
  "durationMinutes": 60,
  "mode": "focus"
}
```

联调版会记录守门开始和结束时间，并在查询时计算剩余时间和进度。真正锁应用仍需要 Android 本地服务执行。

## 白名单

- `GET /api/student/whitelist`
- `GET /api/student/whitelist/edit`
- `GET /api/student/apps/search?keyword=微信`
- `POST /api/student/whitelist/add`
- `POST /api/student/whitelist/update`

`POST /api/student/whitelist/update` 请求示例：

```json
{
  "apps": [
    { "appId": "wechat", "allowed": true },
    { "appId": "douyin", "allowed": false }
  ]
}
```

必要应用会返回 `locked: true`，前端应禁用关闭操作。

## 临时放行

- `GET /api/student/release/apps`
- `POST /api/student/release/request`
- `GET /api/student/release/request/:requestId`
- `POST /api/student/release/request/:requestId/cancel`

当前联调版支持提交、查询、撤回。审批通过和拒绝字段已预留，后续接家长端审批。

## 我的与同步

- `GET /api/student/profile`
- `POST /api/student/binding/unbind`
- `POST /api/student/sync`

解绑第一版不会直接解绑，只返回“需要监管人确认”。

## 页面对应

1. 绑定监管人：绑定接口。
2. 权限检查引导：权限接口。
3. 首页：首页总览接口。
4. 守门模式设置：守门状态与开启接口。
5. 守护详情：守护详情接口。
6. 白名单：白名单和搜索接口。
7. 白名单编辑：白名单编辑和保存接口。
8. 权限：权限接口。
9. 申请临时放行：可申请应用和提交接口。
10. 审批结果：申请详情和撤回接口。
11. 我的：个人设备信息接口。
