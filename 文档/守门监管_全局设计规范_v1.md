# 守卫监管 App 全局设计规范

版本：v1.0  
适用范围：守卫监管 PWA / H5 原型 / 移动端 Web App  
设计基调：温和、可信、安全、家庭守护、轻拟物、东方雅致科技感

---

## 1. 总体设计方向

### 1.1 核心气质

本项目不是传统后台管理系统，也不是强管控、冷冰冰的安全软件。它的视觉核心应当是：

- **安全但不压迫**：让家长感觉可靠，让孩子不觉得被强硬监控。
- **温柔但有秩序**：页面要有家庭陪伴感，同时保留清晰的审批、控制、状态反馈。
- **高级但不炫技**：不要强科技蓝光、赛博网格、荧光线条；应使用柔和渐变、低饱和绿色、米白背景、真实图像资产。
- **像一个精致的家庭安全管家 App**：重点不是“锁机”，而是“守护节奏”。

一句话定义：

> 米白暖底 + 墨绿主色 + 轻拟物卡片 + 真实柔和 3D 安全视觉，构成一个温和可信的家庭守护系统。

---

## 2. 视觉关键词

设计时必须围绕以下关键词执行：

- 家庭守护
- 安全边界
- 温和监管
- 低压感
- 可信赖
- 轻拟物
- 留白
- 米白暖色
- 墨绿秩序感
- 柔和光影
- 真实图片资产
- 高级但克制

禁止出现以下风格：

- 强赛博风
- 高饱和蓝紫科技风
- 过重玻璃拟态
- 大面积霓虹光效
- 冷冰冰的企业后台风
- 过度卡通幼稚风
- 图标线条粗糙、像临时拼出来
- 图片素材带未抠干净的方形背景

## 3. 画布与页面基础

### 3.1 移动端画布

推荐主容器宽度：

```css
.app-shell {
  width: min(100vw, 430px);
  min-height: 100vh;
  margin: 0 auto;
}
```

设计应优先适配 390px–430px 宽度的移动端屏幕。

### 3.2 页面背景

背景不是纯白，也不是纯灰。必须使用暖米白渐变，营造柔和、安全、生活化的氛围。

推荐背景：

```css
body {
  background:
    radial-gradient(circle at 12% 4%, rgba(255,255,255,.9), transparent 34%),
    radial-gradient(circle at 82% 12%, rgba(211,229,202,.32), transparent 34%),
    linear-gradient(180deg, #f8f1e5 0%, #f5efe4 45%, #f6f0e6 100%);
}
```

背景要求：

- 整体偏暖，不要冷白。
- 允许有极轻微绿色光晕，但不能抢内容。
- 不要出现明显网格、强纹理、复杂背景图。
- 页面底色应像高级纸张或柔和奶油色，不应像普通网页白底。

## 5. 字体规范

### 5.1 字体组合

全局字体建议：

```css
--ui-font: -apple-system, BlinkMacSystemFont, "SF Pro Display", "SF Pro Text", "PingFang SC", "Microsoft YaHei", sans-serif;
--title-font: "Songti SC", "STSong", "Noto Serif SC", serif;
```

### 5.2 字体风格

本项目标题应使用偏宋体/衬线气质，形成“东方、稳重、守护”的感觉。正文使用系统无衬线字体，保证移动端清晰度。

### 5.3 字号层级

| 类型 | 推荐字号 | 字重 | 用途 |
|---|---:|---:|---|
| 首页主标题 | 42–44px | 800–900 | “守卫监管” |
| 页面标题 | 38–42px | 800–900 | “设备控制”“我的” |
| 模块标题 | 25–28px | 850–900 | “待审批申请”“审批记录” |
| 卡片标题 | 20–24px | 800–900 | 应用名称、设置项 |
| 正文 | 14–16px | 400–600 | 描述、说明 |
| 小标签 | 12–14px | 700–850 | 在线、待审批、已批准 |
| 关键数字 | 38–42px | 850–900 | 待审批数量、正在放行数量 |

### 5.4 字间距

标题类文字应略微增加字间距：

```css
letter-spacing: .04em;
```

首页副标题可更松一点：

```css
letter-spacing: .12em;
```

## 6. 圆角规范

整体使用大圆角，营造柔和、安全、亲和的感觉。

| 场景 | 圆角 |
|---|---:|
| 大卡片 | 26–30px |
| 普通卡片 | 22–26px |
| 内部设置项 | 18–22px |
| 小按钮 | 14–16px |
| 胶囊标签 | 999px |
| 圆形图标 | 50% |

禁止：

- 直角卡片。
- 4px / 6px 这种过硬后台感圆角。
- 所有组件圆角完全一样，导致层次死板。

## 7. 阴影与光影规范

### 7.1 卡片阴影

主卡片阴影应柔和、扩散、低透明度：

```css
box-shadow: 0 18px 50px rgba(70, 51, 25, 0.09);
```

小组件阴影：

```css
box-shadow: 0 10px 26px rgba(70, 51, 25, 0.06);
```

### 7.2 内发光

卡片和按钮可以使用轻微内高光：

```css
box-shadow:
  0 18px 50px rgba(70, 51, 25, .09),
  inset 0 1px 0 rgba(255,255,255,.72);
```

### 7.3 图片阴影

真实图片资产应使用 `drop-shadow`，不能使用生硬黑影。

```css
filter: drop-shadow(0 18px 24px rgba(45, 39, 30, .10));
```

图片阴影必须满足：

- 阴影偏暖，不要纯黑。
- 阴影透明度低。
- 不能有明显描边。
- 手机、盾牌、头像、图标都应有轻微落地感。

## 8. 卡片系统

### 8.1 基础卡片

```css
.card {
  background: rgba(255,252,244,.86);
  border: 1px solid rgba(70,81,58,.1);
  box-shadow: 0 18px 50px rgba(70,51,25,.09);
  border-radius: 28px;
  backdrop-filter: blur(16px);
}
```

卡片设计原则：

- 卡片应有轻微透气感，但不能透明到影响阅读。
- 边框必须非常轻，偏绿灰或暖灰。
- 卡片之间需要足够间距，不要堆得太紧。
- 内部重要区域可以再套一层浅色子卡片。

### 8.2 页面主卡片

例如设备信息卡、我的信息卡，应使用：

- 高度 170–220px
- 左图右文结构
- 大圆角 28–30px
- 背景带轻微径向光斑
- 右侧允许有真实盾牌/安全模型装饰

### 8.3 列表卡片

例如审批记录、设置项：

- 整体卡片外层圆角 26–28px。
- 列表项高度 68–96px。
- 分割线使用极浅暖灰，不要深灰线。
- 右箭头颜色为弱灰棕，不要黑色。

## 9. 首页规范

首页是最重要的情绪页，必须传递“今天也在帮你守住节奏”的安全感。

### 9.1 顶部品牌区

结构：

- 左侧：品牌名称“守卫监管” + 小盾牌标识。
- 下方：副标题“今天也在帮你守住节奏”。
- 右侧：刷新、设置两个圆形按钮。

要求：

- “守卫监管”使用大号宋体，墨绿色。
- 小盾牌不能太大，起点缀作用。
- 刷新、设置按钮应是圆形轻拟物按钮，带轻微内高光。
- 按钮文字放在图标下方，字号 13–14px。

### 9.2 首页主视觉区

文案：

- 主标题：`用心守护 · 放心成长`
- 副标题：`科技守护边界，陪伴孩子健康成长`
- 状态：`系统运行正常`

主视觉要求：

- 右侧必须使用真实感安全模型图，而不是纯 CSS 临时画图。
- 图像应包含：盾牌、手机、圆形球体、柔和椭圆底座、环绕线。
- 图片必须为透明背景 PNG，不能带方形底色。
- 图像边缘要干净，不能出现未抠干净的白色矩形。
- 图像整体应融入背景，不能像硬贴上去。

推荐图片处理方式：

1. 从参考图或生成图中裁出主视觉。
2. 去除背景，导出透明 PNG。
3. 嵌入 HTML 时使用 `img`，不要再用 CSS 强行画复杂 3D 模型。
4. 使用 `mix-blend-mode: multiply` 或透明 PNG 融合背景。
5. 加轻微 `drop-shadow`。

### 9.3 首页统计卡片

两张统计卡片：

- 待审批
- 正在放行

布局：

```text
[圆形真实图标]  [标题]
              [数字]
              [单位]
```

规范：

- 外层是一个白色半透明大卡片，包含两个小卡片。
- 两个小卡片背景要有轻微差异，但整体统一。
- 左侧图标必须使用圆形真实质感图标，不建议纯线性图标。
- 数字必须使用墨绿大字。
- 单位文字要弱化。

图标要求：

- 待审批：剪贴板/申请单图标，绿色圆底。
- 正在放行：沙漏/时间图标，偏鼠尾草绿圆底。
- 图标要居中，不能偏上或偏下。
- 图标不能糊，不能有多余背景块。

### 9.4 待审批申请卡片

结构：

- 应用图标
- 应用名称
- 申请信息
- 更多按钮
- 两个批准按钮
- 查看详情按钮

按钮规范：

- 次按钮：白底绿边。
- 主按钮：墨绿渐变。
- 主按钮应比次按钮更有视觉重量。
- 按钮圆角 14–16px。
- 按钮高度 46–50px。

### 9.5 正在放行卡片

结构：

- 左侧应用信息
- 右侧圆环倒计时

倒计时圆环规范：

- 使用墨绿进度 + 浅绿底圈。
- 中心白色或浅米色内圈。
- 数字大，单位小。
- 不要使用过亮颜色。

## 10. 设备页规范

设备页的目标是让用户快速确认设备状态，并能执行保护控制。

### 10.1 页面标题区

- 标题：设备控制
- 右侧：通知按钮
- 标题居中，使用墨绿宋体。
- 通知按钮使用圆形浅底，不要强提示色。

### 10.2 设备信息主卡片

结构：

```text
左侧：完整手机模型 + 底座
右侧：设备名称 + 在线标签
     最近操控时间
     设备标识
     后台保护运行中
右侧背景：盾牌安全模型装饰
```

关键要求：

- 左侧手机必须完整显示，不能只露上半部分。
- 手机图片必须是透明背景，不能有方形底图。
- 如果手机素材来自截图裁切，必须抠图干净。
- 手机底部应保留底座或落地阴影，增强真实感。
- 右侧盾牌装饰不能遮挡文字。
- 盾牌可以轻微透明，融入背景。

设备手机图片规范：

```css
.device-phone-real {
  position: absolute;
  left: -2px;
  top: -2px;
  width: 118px;
  height: 180px;
  object-fit: contain;
  object-position: center center;
  mix-blend-mode: multiply;
  filter: drop-shadow(0 18px 24px rgba(45,39,30,.10));
  z-index: 4;
}
```

如果出现方形底图，必须重新处理：

- 用透明 PNG 替换。
- 删除背景色接近 `#f8f1e5` 的区域。
- 检查四周是否有残留矩形。
- 不能通过简单调 opacity 掩盖问题。

### 10.3 状态标签

在线标签：

```css
.online {
  color: #0f513d;
  background: rgba(221,238,226,.84);
  border-radius: 999px;
  padding: 6px 11px;
  font-size: 13px;
  font-weight: 850;
}
```

状态标签应简洁、低饱和，不要像系统警告。

### 10.4 保护控制卡片

结构：

- 标题：保护控制
- 设置项：后台保护
- 开关
- 重新锁紧按钮
- 说明文字

规范：

- “后台保护”使用墨绿大字。
- 说明文字使用灰棕色。
- 开启状态的开关使用墨绿渐变。
- 危险按钮使用红色文字和浅红边框，不能填充大红背景。

危险按钮：

```css
.danger-button {
  background: rgba(255,255,255,.35);
  border: 1px solid rgba(217,87,87,.38);
  color: #d95757;
}
```

### 10.5 高风险入口

- 标题使用橙色。
- 图标使用橙色线性警告图标。
- 内容卡片仍然保持白色/浅米色，不要整块橙色。

### 10.6 桌面图标设置

- 标题图标为四宫格。
- 功能描述要短。
- 开关关闭状态使用米灰色。

### 10.7 最近远程命令

命令列表应呈现为安静的记录卡片：

- 左侧浅绿圆形图标。
- 中间：状态小标签 + 命令名称 + 时间。
- 右侧：箭头。
- 不要做成强提醒消息。

## 11. 我的页规范

我的页强调“监管人身份”和“家庭守护者”的温和感。

### 11.1 顶部标题

- 标题：我的
- 居中。
- 右侧设置按钮。

### 11.2 监管人信息卡片

结构：

```text
左侧：家庭头像
右侧：监管人账号
     守卫监管
     你是孩子成长路上的守护者
```

头像规范：

- 使用圆形家庭头像图。
- 风格应是柔和 3D / 轻卡通拟物。
- 不要使用真实人物照片。
- 不要使用普通默认头像。
- 头像背景必须透明或自然圆形，不要有方形底图。
- 可以附带小盾牌，强化守护身份。

头像 CSS：

```css
.avatar-photo-real {
  width: 102px;
  height: 102px;
  object-fit: cover;
  border-radius: 50%;
  filter: drop-shadow(0 16px 26px rgba(85,64,36,.12));
  mix-blend-mode: multiply;
}
```

### 11.3 审批记录

审批记录应使用列表卡片：

- 每条记录高度约 96px。
- 左侧应用图标。
- 中间状态标签、标题、说明。
- 右侧箭头。

应用图标规范：

- 王者荣耀：保留橙色 `5v5` 视觉，带柔和高光。
- 智能助理：绿色圆角图标，简洁两点/对话气泡感觉。

### 11.4 设置组

设置组包括：

- 修改密码
- 刷新请求
- 退出登录

规范：

- 每项高度 68–74px。
- 字重 800 左右。
- 右箭头弱化。
- 分割线极浅。
- 不要给每一项单独加厚边框。

### 11.5 添加到桌面卡片

结构：

- 左侧：盾牌装置图。
- 右侧：标题、说明、按钮。

左侧图标要求：

- 必须与首页主视觉一致：盾牌、底座、球体、环绕线。
- 背景透明。
- 不能带方形白底。
- 尺寸约 100–120px。

按钮：

- 墨绿渐变。
- 圆角 14px。
- 高度 42–44px。

## 12. 图像资产规范

本项目视觉质感很大程度依赖图像资产。复杂的 3D 图标不建议用 CSS 硬画，应使用精修图片。

### 12.1 图片类型

建议准备以下图片资产：

| 文件 | 用途 |
|---|---|
| `hero_art.png` | 首页右侧主视觉 |
| `stat_clipboard.png` | 待审批统计图标 |
| `stat_hourglass.png` | 正在放行统计图标 |
| `device_phone.png` | 设备页左侧手机模型 |
| `device_shield.png` | 设备页右侧盾牌装饰 |
| `family_avatar.png` | 我的页家庭头像 |
| `install_visual.png` | 添加到桌面卡片图标 |

### 12.2 图片处理要求

所有图片必须满足：

- PNG 格式。
- 背景透明，或与页面背景自然融合。
- 不得出现白色/米色方形底块。
- 边缘干净，没有裁切残留。
- 阴影柔和，不要黑边。
- 图片大小要足够清晰，但不要过度锐化。

### 12.3 抠图检查清单

每次导入图片前必须检查：

- 放到深色背景上是否有白边。
- 放到页面背景上是否出现矩形底色。
- 图片边缘是否有半透明脏边。
- 物体是否被裁掉，比如手机是否缺下半部分。
- 阴影是否自然。
- 图片是否和周围文字发生遮挡。

### 12.4 什么时候用图片，什么时候用 CSS

应使用图片的场景：

- 3D 盾牌。
- 手机模型。
- 家庭头像。
- 复杂拟物图标。
- 带柔和光影的装置插画。

应使用 CSS/SVG 的场景：

- 简单线性图标。
- 箭头。
- 设置图标。
- 开关。
- 按钮。
- 进度圆环。

原则：

> 复杂质感用图片，简单交互用 CSS/SVG。

## 18. 交互动效规范

动效要轻，不要炫。

### 18.1 页面切换

```css
@keyframes pageIn {
  from { opacity: 0; transform: translateY(10px); }
  to { opacity: 1; transform: translateY(0); }
}
```

页面切换应在 0.28–0.35s 内完成。

### 18.2 点击反馈

按钮点击：

```css
transform: scale(.96);
```

要求：

- 点击反馈轻微缩放。
- 不要强烈弹跳。
- 不要复杂过渡。

### 18.3 Toast

Toast 应位于底部导航上方。

- 背景深绿黑。
- 白色文字。
- 圆角 16px。
- 显示时间约 1.8s。

## 24. 复刻参考图的方法论

要接近参考图，不要只靠 CSS 硬画。正确流程如下：

### 第一步：先确定页面结构

先把 HTML 页面结构搭好：

- 顶部标题
- 主视觉区
- 统计卡片
- 申请卡片
- 设备信息卡
- 设置组
- 我的页信息卡

### 第二步：用 CSS 做整体骨架

CSS 负责：

- 页面背景
- 卡片形状
- 字体
- 间距
- 按钮
- 标签
- 开关
- 导航

### 第三步：复杂视觉改用图片资产

复杂视觉包括：

- 3D 盾牌
- 手机模型
- 家庭头像
- 圆形统计图标
- 添加到桌面装置图

这些不要用 CSS 勉强复刻，否则会显得假。

### 第四步：抠图与融合

图片导入后必须检查：

- 是否完整。
- 是否透明。
- 是否带方形背景。
- 是否和页面背景融合。
- 是否需要 `mix-blend-mode: multiply`。
- 是否需要 `drop-shadow`。

### 第五步：局部反复微调

每次只修一个问题：

- 手机缺下半部分，就只换手机图。
- 多出半个手机，就只删重复图片。
- 有方形底图，就只重新抠图。
- 图标不真实，就只换图标资产。

这样比大范围重写更稳定。

## 25. 最终视觉标准

完成后的页面应该达到以下状态：

- 第一眼像一个完整成熟的移动端 App，而不是临时网页。
- 页面温柔、干净、可信，不像强制管控工具。
- 所有图片都像自然融入页面，没有硬贴感。
- 首页上半部分最精致，是整个产品的视觉门面。
- 设备页手机必须真实、完整、干净。
- 我的页头像必须有家庭守护感。
- 整体不需要强科技感，但要有高级的安全感。

最终判断标准：

> 如果把页面截图发给别人，对方应该觉得这是一个“已经设计过的 App 界面”，而不是“AI 随便生成的网页”。

## 0. 关键视觉资产硬规则

守卫监管后续前端美化中，决定高级感的关键部位不能再主要依赖 CSS 手绘。

必须优先使用真实图片、生成图片、设计稿导出的位图或高质量 SVG 资产的区域：
- 首页主视觉：手机、盾牌、环绕线、家庭守护场景。
- 设备页手机模型。
- 我的页监管人/家庭头像。
- 添加到桌面提示图。
- 统计卡核心图标。

CSS 的职责是排版、留白、卡片质感、状态反馈、按钮、开关和轻动效；CSS 不再承担复杂插画和真实设备图的绘制。只有极简单的符号、线性小图标、背景光晕可以用 CSS/SVG 直接画。
