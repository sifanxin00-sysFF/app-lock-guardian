# 番茄 ToDo 功能拆解与复刻方案

本文用于把“番茄 ToDo 为什么锁得住”拆清楚，再反推守门应用锁应该怎么做。DeepSeek 只作为普通测试样例，不是产品目标。

## 一句话结论

番茄 ToDo 的核心不是 Device Owner，也不是单个 App 弹窗锁。它更像一个“本地锁机系统”：锁机开始后，番茄自己的锁机页成为主状态；用户打开白名单外 App 时，系统底层可能已经切过去，但番茄会用无障碍、悬浮窗、前台服务和前台识别把用户拉回锁机页，所以用户感觉“退不出去”。

守门应用锁要复刻的核心效果是：本地或网页启动守门模式后，白名单外 App 不能使用，用户会被拉回守门页；网页端再额外提供远程启动、结束、白名单配置、隐藏图标和系统设置锁定控制。

## 权限与能力总表

| 权限 / 能力 | 番茄 ToDo 用来做什么 | 守门应用锁怎么用 |
|---|---|---|
| 使用情况访问 `PACKAGE_USAGE_STATS` | 读取当前前台 App，判断是否白名单外 | 作为稳定兜底识别前台包 |
| 无障碍服务 | 监听窗口变化、内容变化、通知变化，必要时执行返回/拉回 | 作为第一拦截入口，收到白名单外 App 事件立刻拉回守门页 |
| 悬浮窗 `SYSTEM_ALERT_WINDOW` | 在其他 App 上方显示覆盖层或提示层 | 作为辅助阻挡，不再作为唯一锁机方式 |
| 前台服务 `FOREGROUND_SERVICE` | 锁机期间保持后台服务常驻 | 守门模式期间必须常驻通知，持续心跳和命令轮询 |
| 通知权限 | 展示不可清除的运行通知 | 显示“守门模式运行中”，并提示剩余时间 |
| WakeLock | 降低锁机期间被休眠影响 | 后台保护服务运行时持有 partial wakelock，并在诊断显示状态 |
| 忽略电池优化 | 降低系统杀后台概率 | 权限页明确引导用户开启电池无限制 |
| 开机自启动 / 后台运行 | 小米系统保活 | 权限页做小米专项引导，无法自动开启时写清手动路径 |
| 最近任务加锁 | 小米上防止被清后台 | 权限页写成必须手动操作项 |
| 查询安装应用 `QUERY_ALL_PACKAGES` | 展示白名单和限制名单 | 应用选择页显示所有可选 App，支持搜索 |
| 隐藏桌面图标 | 防卸载、防用户从桌面入口关闭 | 只作为强制性增强项，由网页或本地明确开关触发 |
| 设备管理员 | 增加卸载阻力 | 保留为防卸载辅助，不作为锁机主能力 |
| 动态壁纸 / 设置壁纸 | 番茄可能用来辅助桌面加固和保活 | 暂不作为第一期必做；先记录为高级备选 |
| 通知监听 | 番茄可能用于屏蔽通知干扰 | 暂不作为第一期必做；后续需要时再加 |

## 功能拆解

### 1. 锁机 / 学霸模式

用户看到的效果：启动 30 分钟、1 小时、2 小时后，手机进入专注状态。白名单外 App 打开后，用户会被拉回锁机页，不能继续使用。

番茄 ToDo 的实现步骤大概是：

1. 用户启动锁机，保存结束时间和模式。
2. 启动前台服务，显示常驻通知。
3. 无障碍服务监听窗口变化，UsageStats 兜底读取前台 App。
4. 识别到白名单外包名后，启动或拉回番茄锁机页。
5. 锁机页显示剩余时间，不给普通关闭入口。

守门应用锁实现方式：

1. 新增或强化 `GuardModeActivity`，作为锁机期间的主页面。
2. `AppLockStore` 保存守门模式开关、结束时间、模式类型、白名单。
3. `LockAccessibilityService` 和 `AppMonitorService` 都调用统一判断：当前包是否应该被守门模式拦截。
4. 命中拦截时优先拉回 `GuardModeActivity`，原锁窗只做备用。
5. 到期后自动退出守门模式。

失败提示：如果没拦住，要能显示是无障碍未开启、使用情况访问不可用、后台服务未运行、前台识别失败，还是目标被白名单放行。

### 2. 白名单

用户看到的效果：电话、短信、桌面、守门应用锁、监管网页浏览器可以正常用；其他 App 根据模式被拦。

番茄 ToDo 的实现步骤大概是：

1. 保存一组允许包名。
2. 锁机期间每次前台变化都查包名是否在白名单。
3. 电话、短信这类系统关键应用默认放行。

守门应用锁实现方式：

1. `AppLockStore` 保存白名单集合。
2. 应用选择页增加“加入白名单 / 移出白名单”。
3. 默认白名单包含守门应用锁、桌面、电话、短信、必要系统组件、监管网页浏览器。
4. DeepSeek 不默认放行，也不特殊处理。

失败提示：诊断里记录“本次放行原因：命中白名单 / 系统关键应用 / 守门应用锁自身”。

### 3. 防退出

用户看到的效果：锁机期间打开番茄 ToDo 自己，也不能回到普通首页关闭锁机。

番茄 ToDo 的实现步骤大概是：

1. 锁机期间监听到自己 App 被打开。
2. 如果不是锁机页，就重定向到锁机页。
3. “禁止打开主界面”作为强制性选项展示给用户。

守门应用锁实现方式：

1. 守门模式有效时，`MainActivity` 启动先检查模式。
2. 如果守门模式未到期，直接跳转 `GuardModeActivity`。
3. 普通首页只在未锁机或管理员/监管放行后可进入。

失败提示：如果用户进入了普通首页，诊断记录“守门模式期间主界面未被拦截”。

### 4. 自动防关闭

用户看到的效果：想进设置、应用管理、权限管理、小米安全中心关闭 App 时，会被拦回。

番茄 ToDo 的实现步骤大概是：

1. 维护一组高风险包名和页面关键字。
2. 无障碍检测到系统设置、应用管理、权限页、卸载页时，拉回锁机页。
3. 这个能力作为“强制性选项”，不是普通提醒。

守门应用锁实现方式：

1. 保留现有“锁系统设置和卸载入口”策略。
2. 默认关闭，避免配置阶段误伤。
3. 网页端或本地强制性设置明确开启后，守门模式期间拦截系统设置、应用管理、权限管理、小米安全中心、卸载入口。

失败提示：诊断记录“高风险入口命中 / 未开启高风险策略所以放行”。

### 5. 悬浮窗 / 覆盖层

用户看到的效果：某些情况下会看到覆盖提示，或者感觉页面被挡住。

番茄 ToDo 的实现步骤大概是：

1. 拿悬浮窗权限。
2. 锁机期间创建覆盖层，用于提示、阻挡或辅助检测。
3. 覆盖层不是唯一手段，真正核心仍是拉回锁机页。

守门应用锁实现方式：

1. 保留 `LockOverlayController`，但降级为辅助。
2. 守门模式主体验改为 `GuardModeActivity` 拉回。
3. 覆盖层添加失败不能导致崩溃，只写诊断。

失败提示：诊断记录“悬浮窗权限未开 / 覆盖层添加失败 / 已改用守门页拉回”。

### 6. 后台保活

用户看到的效果：锁机开始后，通知栏有常驻通知，清后台后仍尽量继续锁。

番茄 ToDo 的实现步骤大概是：

1. 启动前台服务。
2. 显示不可清除通知。
3. 持有 WakeLock。
4. 引导开启电池无限制、自启动、后台运行。

守门应用锁实现方式：

1. `AppMonitorService` 在守门模式期间必须作为前台服务运行。
2. 通知文案显示“守门模式运行中，剩余 xx 分钟”。
3. 心跳每轮刷新，远程命令轮询不能卡住前台识别。
4. 首页显示后台保护正常、疑似冻结、无障碍疑似停止响应。

失败提示：首页和诊断显示最近后台事件、心跳停滞秒数、WakeLock 状态。

### 7. 小米专项

用户看到的效果：权限页明确告诉用户小米手机必须做哪些设置。

番茄 ToDo 的实现步骤大概是：

1. 提示“小米机型请务必先在多任务界面加锁”。
2. 引导自启动、后台运行、电池无限制、悬浮窗、无障碍。
3. 无法自动打开的能力只做手动说明。

守门应用锁实现方式：

1. 权限页新增“小米专项”区域。
2. 每项都写清楚：为什么要开、怎么开、当前能否检测。
3. 最近任务加锁无法可靠自动检测时，提供“一键复制检查清单”。

失败提示：如果后台心跳和无障碍心跳同时停住，首页显示“疑似被系统冻结，请检查小米专项设置”。

### 8. 隐藏图标

用户看到的效果：桌面上找不到 App 图标，降低卸载和关闭概率。

番茄 ToDo 的实现步骤大概是：

1. 通过启用/禁用 launcher activity 或别名隐藏图标。
2. 作为防卸载增强项，用户需要明确打开。

守门应用锁实现方式：

1. 保留现有网页 `hide_launcher` / `show_launcher`。
2. 本地也可以放在“强制性选项”里，但必须写清风险。
3. 开启守门模式不自动隐藏图标。

失败提示：诊断记录期望图标状态、实际状态、最近一次图标命令结果。

### 9. 远程控制

用户看到的效果：监管网页可以远程让手机开始锁机、结束锁机、调整白名单。

番茄 ToDo 没有这一层，这是守门应用锁要新增的能力。

守门应用锁实现方式：

1. 网页端设备卡片新增“守门模式”分组。
2. 命令包括：锁机 30 分钟、锁机 1 小时、锁机 2 小时、结束守门模式。
3. 后续增加白名单配置命令。
4. Android 端收到命令后写入本地守门模式状态，并回传执行结果。

失败提示：网页命令记录显示“已开始 / 已结束 / 失败原因 / 剩余时间 / 当前白名单数量”。

## 第一阶段推荐执行顺序

1. 先做 `GuardModeActivity`：锁机期间统一显示守门页。
2. 改拦截主逻辑：白名单外 App 不再优先弹锁窗，而是拉回守门页。
3. 改首页和权限页：明确守门模式、白名单、小米专项、强制性选项。
4. 接网页远程启动：30 分钟、1 小时、2 小时、结束守门模式。
5. 最后做真机验收：白名单外拉回、白名单内放行、到期恢复、远程命令生效。

## 当前不做

- 不做 Device Owner。
- 不清手机数据。
- 不把 DeepSeek 写死。
- 不复制番茄 ToDo 代码。
- 不默认隐藏桌面图标。
- 不默认锁系统设置；系统设置锁定必须由网页或强制性选项开启。
