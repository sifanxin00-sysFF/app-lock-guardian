# AppLockGuard

半强管控版 Android 应用锁，适合已经长期使用、不方便恢复出厂设置的个人手机。

当前 MVP 能力：

- 设置保管密码，两次输入确认，只保存 PBKDF2 哈希。
- 从已安装应用中选择黑名单。
- 常驻前台服务通过 Usage Access 检测前台应用。
- 无障碍服务作为兜底监测入口。
- 打开被锁应用时进入全屏锁定页。
- 输入正确密码后临时放行，默认 10 分钟。
- 设置页、修改密码、暂停保护需要保管密码。
- 开机后自动恢复保护服务。
- 可启用 Android 设备管理员，增加卸载阻力。

重要边界：

- 这是半强管控版，不是 Device Owner / MDM。
- 不能保证完全阻止卸载、清除数据、强行停止或关闭权限。
- 红米/小米系统后台策略较强，必须手动开启自启动和电池无限制。
- 如果要做到真正系统级防绕过，需要恢复出厂设置后配置 Device Owner。

## 构建

当前工作机未安装 Java、Android SDK、Gradle 或 ADB，所以这里没有生成 APK。

在你的开发机上：

1. 安装 Android Studio。
2. 打开本目录：`D:\AI_Test\AppLockGuard`。
3. 等待 Android Studio 同步 Gradle。
4. 连接手机后点击 Run，或使用 `Build > Build APK(s)`。

## 红米 K80 至尊版授权建议

安装后，打开应用里的“权限与红米设置引导”，按顺序开启：

1. 使用情况访问：允许本应用读取前台应用变化。
2. 无障碍监测服务：用于更快检测被锁应用启动。
3. 通知权限：保持前台服务通知可见。
4. 忽略电池优化：降低后台被杀概率。
5. 自启动：允许重启后恢复保护。
6. 电池策略：建议设置为无限制。
7. 设备管理员：增加卸载阻力。

HyperOS / MIUI 不同版本的菜单名字可能会变化。如果按钮无法直达对应页面，就从系统设置里搜索：

- 使用情况访问权限
- 无障碍
- 自启动
- 电池优化
- 应用信息
- 设备管理应用

## 使用流程

1. 首次打开，设置保管密码。
2. 把密码交给另一个人保管。
3. 打开权限引导，完成手机授权。
4. 进入“选择要锁定的应用”，勾选游戏、短视频、社交应用。
5. 点击“开启保护”。
6. 尝试打开被锁应用，应该出现锁定页。
